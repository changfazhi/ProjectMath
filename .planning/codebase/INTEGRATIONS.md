# External Integrations

**Analysis Date:** 2026-06-26

## APIs & External Services

**Google Gemini (AI):**
- SDK: `@google/genai` ^2.9.0
- Client: `backend/src/db/gemini.ts` — lazy singleton via `getGemini()`; throws on missing key rather than crashing at boot
- Default model: `gemini-2.5-flash` (overridable via `GEMINI_MODEL` env var)
- Auth: `GEMINI_API_KEY` environment variable (backend only — never exposed to browser)
- Used by:
  - `backend/src/routes/chat.ts` → `backend/src/services/chatService.ts` — Socratic hint chatbot
  - `backend/src/routes/grade.ts` → `backend/src/services/gradingService.ts` — handwritten solution grading
- Structured output (deterministic JSON schema) used in grading: `gradable`, `rejection_reason`, `ignored_images`, `parts[{label, verdict, marks_awarded, marks_total, errors, hints, summary}]`, `overall_feedback`

## Data Storage

**Databases:**
- Supabase (PostgreSQL)
  - Client: `backend/src/db/supabase.ts` — `createClient(SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY)`
  - Service role key used (full access, backend-only)
  - Tables used by routes:
    - `topics` — `backend/src/routes/topics.ts`
    - `questions` — `backend/src/routes/questions.ts`
    - `attempts` — `backend/src/routes/attempts.ts`
    - `topic_concepts` — `backend/src/routes/concepts.ts`
    - `starred_questions` — `backend/src/routes/stars.ts`
    - `chat_messages` — `backend/src/routes/chat.ts`
    - `gradings` — `backend/src/routes/grade.ts`
  - Migrations: `backend/supabase/migrations/001_*.sql` through `017_ri_prelim_2025.sql`
  - All tables require `GRANT ALL ON TABLE public.<table> TO anon, authenticated, service_role;` after creation

**File Storage:**
- Supabase Storage bucket: `solution-uploads` (private)
  - Created in migration `013_solution_gradings.sql`
  - Used by `backend/src/services/gradingService.ts` — images uploaded after successful grading
  - Images arrive via multer (`backend/src/routes/grade.ts`), sent to Gemini as base64 `inlineData`, then stored

**Caching:**
- None (no Redis or in-memory cache beyond the Socket.IO pairing `Map`)

## Authentication & Identity

**Auth Provider:**
- None — no user accounts
- Session identity: UUID v4 in `localStorage` key `session_id`, generated by `frontend/src/lib/session.ts`
- Pairing tokens (phone upload): `crypto.randomBytes(32).base64url()` — unguessable capability tokens, stored in-memory `Map` in `backend/src/services/pairService.ts` (TTL: `PAIR_TTL_MIN`, default 10 min)

## Real-Time: Socket.IO

**Backend setup:** `backend/src/realtime.ts`
- `initRealtime(httpServer)` attaches `socket.io` Server to the Node `http.Server`
- CORS: `origin: '*'`
- `emitToPair(token, event, payload)` — helper used by services to emit to a token room without importing the server directly

**Rooms:** Each pairing token is a Socket.IO room. Desktops join via `pair:subscribe { token }`.

**Event names:**

| Direction | Event | Payload | Description |
|-----------|-------|---------|-------------|
| Client → Server | `pair:subscribe` | `{ token }` | Desktop joins token room |
| Client → Server | `pair:unsubscribe` | `{ token }` | Desktop leaves room |
| Server → Client | `pair:phone-connected` | — | Phone opened the mobile page |
| Server → Client | `pair:image` | image data | Photo streamed from phone to desktop |
| Server → Client | `pair:grading` | — | Grading started |
| Server → Client | `pair:graded` | grading result | Grading complete |
| Server → Client | `pair:error` | error info | Grading failed / rejected |

**Frontend client:** `frontend/src/lib/socket.ts` — lazy singleton `getSocket()` using same-origin `io()` (no URL), tunneled through Vite proxy in dev.

**Frontend hooks:**
- `usePairSocket` — subscribes to token room, forwards events into `usePracticeSession` via `beginExternalGrading` / `receiveGrading` / `rejectExternalGrading`

**Vite proxy (dev only):**
```ts
// frontend/vite.config.ts
'/socket.io': { target: 'ws://localhost:3001', ws: true, changeOrigin: true }
```

## MathLive (Math Input Web Component)

**Package:** `mathlive` ^0.110.0
**Type:** Native web component (`<math-field>`)
**React wrapper:** `frontend/src/components/MathField.tsx`
- Wraps `<math-field>` element using a `ref`
- Exposes `insert(latex, options?)`, `getValue()`, `focus()` as imperative API
- Suppresses built-in virtual keyboard and hamburger menu
- Companion `frontend/src/components/MathKeyboard.tsx` — custom 10-group symbol panel; uses `onMouseDown` (not `onClick`) to avoid stealing focus from the math field

**Insert convention:**
- Template placeholders: `#?` for tab-stops
- Call `mf.insert(latex, { selectionMode: 'placeholder' })` to land cursor inside inserted template
- Compact notation (`\frac34`) auto-expanded by `normalizeLaTeX()` in `backend/src/services/attemptService.ts`
