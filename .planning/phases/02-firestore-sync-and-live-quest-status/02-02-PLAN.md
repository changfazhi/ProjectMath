<objective>
Make the sidebar's quest statuses (✓ correct / ↩ attempted / numbered pending) refresh live when a student returns to the app from a practice question — without a full page reload. Status continues to be derived exclusively from `GET /api/attempts`; no status is ever written to Firestore or any other store.

Purpose: Phase 2 success criteria 3 and 4 — live status updates on return from practice, with the attempts API as the single source of truth.
Output: A focus/visibility-driven re-fetch in `useStudyPlan` that re-derives status from the attempts API while the sidebar is mounted.
</objective>

## Phase Goal

**As a** student working through my daily quests, **I want to** see a quest flip to ✓ in the sidebar as soon as I come back from solving it, **so that** my plan reflects my progress without me having to reload the whole app.
