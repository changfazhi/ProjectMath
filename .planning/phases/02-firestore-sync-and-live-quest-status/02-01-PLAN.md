<objective>
**As a** student who just answered a practice question, **I want** the sidebar to show my updated quest status (‚úì correct or ‚Ü© attempted) when I navigate back to any page, **so that** I can see my progress without reloading the page.

Currently `useStudyPlan` loads quest statuses once ‚Äî when `isOpen` changes from false to true. After that, statuses are stale until the user closes and reopens the sidebar. This plan adds `useLocation().key` as a dependency so the status derivation re-runs on every navigation event while the sidebar is open.

**Implementation**: In `useStudyPlan.ts`, import `useLocation` from `react-router-dom`. Add `location.key` as a dependency to the existing `useEffect`. The effect already short-circuits the plan fetch via `loadStoredPlan()` (localStorage, synchronous, O(1)) and only calls the network for `api.attempts.list()`. Re-running the full effect on navigation is therefore cheap: one localStorage read + one GET /api/attempts.

No other files need changing. The sidebar component, QuestItem, and progress summary all derive from `quests` state, which will reflect the refreshed statuses automatically.
</objective>
