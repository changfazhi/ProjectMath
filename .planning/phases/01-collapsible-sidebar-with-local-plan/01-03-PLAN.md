<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: End-to-end human verification of the Phase 1 sidebar</name>
  <action>No code changes. The human runs the app and walks the seven checks below against the ROADMAP Phase 1 Success Criteria and the UI-SPEC, then approves or reports issues.</action>
  <what-built>A persistent, collapsible study-plan sidebar mounted in RootLayout: a fixed blue trigger tab on every page, a 280px sliding panel listing today's quests with correct/attempted/pending icons and a progress summary, click-to-navigate quests, plus empty / stale-day / loading / error states and full keyboard accessibility.</what-built>
  <how-to-verify>
    Start the app: in one terminal `cd backend && npm run dev`, in another `cd frontend && npm run dev`, then open the printed local URL.

    1. SIDE-01 / SIDE-03 / SIDE-06: On the roadmap page, confirm the blue trigger tab is visible at left-center and the page content is NOT pushed or covered. Navigate to /history, /starred, /stats, and a /practice/:topicId page — the tab is present on each. Default state is collapsed.
    2. SIDE-02 / QUEST-01 / QUEST-02 / QUEST-04: Ensure a plan exists for today (visit /study-plan once to generate, or generate from /review). Click the trigger tab — the panel slides in, titled "Today's Plan", with a "N / M complete" summary + progress bar and a row per quest showing question name, topic name, and a status icon (✓ correct / ↩ attempted / numbered pending).
    3. QUEST-03: Click a quest — you land on /practice/:topicId?question_id=... for that question and the panel stays open.
    4. Refresh-on-open: Answer that question correctly, return, and reopen the panel — its icon is now ✓ (no full reload needed).
    5. SIDE-04 (empty): In DevTools, remove the `study_plan_v1` localStorage key, reload, open the panel — it shows "No plan yet" and a "Generate plan" button that navigates to /review.
    6. SIDE-05 (stale): In DevTools, edit `study_plan_v1` so its `date` is yesterday, reload, open the panel — the quest list is greyed out and an amber "New day" banner with a "Generate plan" button (→ /review) appears; no progress summary.
    7. Accessibility: Open the panel — focus lands on the × close button. Press Escape — the panel closes and focus returns to the trigger tab. Tab through the quest rows and activate one with Enter. Toggle dark mode and confirm every state still reads correctly.
  </how-to-verify>
  <resume-signal>Type "approved" if all seven checks pass, or describe any issues (which step, what you saw vs. expected) so a gap-closure plan can be created.</resume-signal>
