<objective>
**As a** student, **I want to** open a persistent sidebar from any page and see today's study plan with live status, **so that** I can jump straight to my next practice question without losing my place.

This is the Walking Skeleton for the sidebar milestone: the thinnest real end-to-end path through the new feature. It extracts the shared localStorage helpers, adds the `useStudyPlan` hook, builds the `QuestItem` and `StudyPlanSidebar` components (happy path: active plan), and mounts the sidebar in `RootLayout`. After this plan a student can open the sidebar on any page, see today's quests with correct/attempted/pending icons and a progress summary, click a quest, and navigate to it.

Purpose: Prove the full sidebar slice works end-to-end before layering on edge-case states.
Output: `lib/studyPlan.ts`, `useStudyPlan` hook, `QuestItem`, `StudyPlanSidebar`, mounted in `App.tsx`; `QuestStatus` added to `types/api.ts`.

Edge-case states (empty / stale-day / loading / error), the full WCAG focus-management, and status-refresh refinement are deferred to Plan 01-02 (which refines these same files). They are intentionally NOT in this skeleton.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → sidebar render | Plan JSON (`study_plan_v1`) is user/locally-controlled; question_name/topic_name strings flow into the DOM |
| quest data → router navigation | topic_id / question_id values are interpolated into a `/practice/...` URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | localStorage `study_plan_v1` JSON | medium | mitigate | `loadStoredPlan()` parses inside try/catch and shape-guards `items` is an array; malformed → `{ plan: null, isStale: false }`, never throws into render |
| T-01-02 | Information Disclosure (XSS) | question_name / topic_name rendered in QuestItem | low | mitigate | Render strings only as React text children (never `dangerouslySetInnerHTML`); React auto-escapes |
| T-01-03 | Tampering | topic_id / question_id interpolated into navigation URL | low | accept | Values originate from the same-origin study-plan API and local plan; navigation targets an internal SPA route only (no external redirect, no eval). Low impact at ASVS L1 |
</threat_model>
