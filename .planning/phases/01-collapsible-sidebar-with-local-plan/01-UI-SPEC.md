# Phase 1 — UI Design Contract

> Visual and interaction contract for the Collapsible Sidebar with Local Plan phase.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | components.json absent; hand-rolled component library |
| Preset | not applicable | no shadcn |
| Component library | Custom (`src/components/ui/`) | Codebase scan |
| Icon library | Inline SVG only | Codebase scan — no lucide/heroicons package |
| Font | Inter (sans), JetBrains Mono (mono) | `tailwind.config.js` fontFamily |
| Dark mode | Tailwind `class` strategy | `tailwind.config.js` darkMode |

Reuse existing components: `Button`, `Card`, `Badge`, `ProgressBar`, `Spinner`. Do not introduce new UI primitives unless listed in the Component Inventory below.

---

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon-to-label gap inside quest item |
| sm | 8px | Quest item internal padding (top/bottom), badge padding |
| md | 16px | Sidebar panel horizontal padding, section gap |
| lg | 24px | Sidebar panel top/bottom padding, header padding |
| xl | 32px | Space between progress summary and quest list |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Quest item row height (min-height) |

Exceptions:
- Sidebar trigger tab: `w-10 h-20` (40 × 80px) — fixed pixel dimensions required for the always-visible affordance; 80px is the only non-standard value; acceptable as a component-specific constant.
- Sidebar panel width: 280px — layout constant, not a spacing token.

## Typography

| Role | Size | Weight | Line Height | Tailwind class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 | 1.5 | `text-sm` | Quest item question name, general copy |
| Label | 12px | 600 | 1.4 | `text-xs font-semibold` | Section headers (uppercase, tracking-wider), topic name under quest item, status count copy |
| Heading | 18px | 700 | 1.2 | `text-lg font-bold` | Sidebar panel title ("Today's Plan") |
| Callout | 16px | 600 | 1.4 | `text-base font-semibold` | Progress summary ("2 / 5 complete"), stale-day banner headline |

Two weights only: 400 (regular) and 600–700 (semibold/bold). No other weights.

Section labels must use `uppercase tracking-wider` to match the existing `QuestionTable` and `TopicDrawer` patterns.

---

## Color

All colors are Tailwind semantic classes. Both light and dark values must be applied on every element.

| Role | Light | Dark | Usage |
|------|-------|------|-------|
| Dominant (60%) | `bg-slate-50` / `bg-white` | `bg-slate-950` | Body background, sidebar panel backdrop |
| Secondary (30%) | `bg-white` | `bg-slate-900` | Sidebar panel surface (same as `Card`) |
| Accent (10%) | `blue-600` | `blue-500` | Trigger tab background, primary CTA button, progress bar fill |
| Success | `emerald-500` | `emerald-400` | Correct quest status icon (✓) |
| Warning | `amber-500` | `amber-400` | Attempted quest status icon (↩) |
| Neutral pending | `bg-slate-200 text-slate-600` | `bg-slate-700 text-slate-300` | Pending quest number badge |
| Stale-day banner | `bg-amber-50 border-amber-200` | `bg-amber-900/20 border-amber-800` | New-day prompt banner |
| Destructive | `red-500` | `red-400` | Error state text only (no destructive actions in Phase 1) |

Accent (`blue-600`) is reserved for:
1. The sidebar trigger tab background
2. The "Generate plan" primary CTA button
3. The quest progress bar fill (`fillClass="bg-blue-500"`)
4. Focus rings (`focus-visible:ring-blue-500`) — inherited from existing `Button`

The progress bar uses `blue-500` (not `emerald-500`) to visually distinguish "daily plan progress" from "topic mastery progress" shown in `TopicDrawer`.

## Layout Contract

### RootLayout change

Current structure:
```
div.min-h-screen.flex.flex-col
  Header (sticky)
  main.flex-1.overflow-hidden.flex.flex-col
    Outlet
```

After Phase 1:
```
div.min-h-screen.flex.flex-col
  Header (sticky)
  div.relative.flex-1.overflow-hidden        ← new wrapper
    main.flex-1.overflow-hidden.flex.flex-col
      Outlet
    StudyPlanSidebar                          ← new sibling
```

The `StudyPlanSidebar` is `position: fixed` and does not affect main content flow.

### Sidebar Trigger Tab

- Position: `fixed left-0` centered vertically at `top-1/2 -translate-y-1/2`
- Dimensions: `w-10 h-20` (40 × 80px)
- Shape: `rounded-r-2xl` (right side rounded, left side flush to viewport edge)
- Background: `bg-blue-600 dark:bg-blue-500 hover:bg-blue-700 dark:hover:bg-blue-400`
- Icon: inline SVG calendar icon, `w-5 h-5 text-white`, centered
- `aria-label`: "Open study plan" (collapsed) / "Close study plan" (expanded)
- Z-index: `z-50`
- When sidebar is expanded: trigger tab remains visible; icon rotates to ← chevron

### Sidebar Panel

- Position: `fixed left-0 top-0 h-full z-40`
- Width: `w-[280px]`
- Background: `bg-white dark:bg-slate-900`
- Shadow: `shadow-2xl`
- Transform: `translate-x-0` when open, `-translate-x-full` when closed
- Animation: `transition-transform duration-300 ease-out`

### Backdrop

- Shown when panel is open on all viewport sizes (same pattern as `TopicDrawer`)
- `fixed inset-0 z-30 bg-black/30 backdrop-blur-sm`
- `transition-opacity duration-300`
- Click backdrop → close panel

### Internal panel layout

```
div.flex.flex-col.h-full
  ── Panel header (px-4 py-5, border-b)
     "Today's Plan" (text-lg font-bold)
     Close × button (right-aligned)
  ── Progress summary (px-4 py-3, border-b)
     "N / M complete" (text-base font-semibold)
     ProgressBar (fillClass="bg-blue-500", size="sm")
  ── Scrollable quest list (flex-1 overflow-y-auto px-4 py-2)
     [QuestItem × N]
  ── Footer (px-4 py-4, border-t) — empty state or stale-day banner
```

## QuestItem Contract

Layout: horizontal flex row, `min-h-[64px] py-3 px-2 rounded-xl hover:bg-slate-50 dark:hover:bg-slate-800 cursor-pointer transition-colors`

```
[status zone 24×24]  [text zone flex-1]
                      Question name (text-sm, truncated at 2 lines)
                      Topic name (text-xs font-semibold text-slate-500 dark:text-slate-400)
```

Status zone rendering (24 × 24px, flex-shrink-0):

| State | Rendering |
|-------|-----------|
| `correct` | SVG check icon `w-5 h-5 text-emerald-500` |
| `attempted` | SVG return-arrow icon `w-5 h-5 text-amber-500` |
| `pending` | `<span>` circle `w-6 h-6 rounded-full bg-slate-200 dark:bg-slate-700 text-xs font-semibold flex items-center justify-center text-slate-600 dark:text-slate-300` with the 1-based ordinal number |

Click: `navigate('/practice/:topicId?question_id=:questionId')`. Sidebar stays open after navigation.

## States Contract

### State 1: Empty (no plan exists)

Shown inside the scrollable body area.

```
[calendar icon, w-10 h-10, text-slate-300 dark:text-slate-600, centered]
"No plan yet"              (text-sm font-semibold, text-slate-500)
"Generate a study plan     (text-sm, text-slate-400, text-center)
to track your daily
practice goals."
[Generate plan button]     (Button variant="primary" size="md" full-width, navigates to /review)
```

No progress bar is shown when no plan exists.

### State 2: Active plan (today's date)

- Progress summary visible at top
- Quest list with `QuestItem` rows
- Footer section: empty (no action needed)

### State 3: Stale plan (plan date ≠ today's date)

- Progress summary hidden
- Quest list rendered with `opacity-40` on all items (the list itself, not individual items)
- Footer section replaced by stale-day banner:

```
div.bg-amber-50.dark:bg-amber-900/20.border.border-amber-200.dark:border-amber-800.rounded-xl.p-3
  "New day"                         (text-sm font-semibold text-amber-800 dark:text-amber-300)
  "Generate today's plan to         (text-xs text-amber-700 dark:text-amber-400)
  continue your streak."
  [Generate plan button]            (Button variant="secondary" size="sm" full-width, navigates to /review)
```

### State 4: Loading

- Spinner centered in scrollable body (`<Spinner size="sm" />`)
- No progress bar
- Trigger tab still visible and clickable (panel can be opened during load)

### State 5: Error

- `text-sm text-red-500` message in scrollable body: "Couldn't load your plan — refresh to try again."
- No retry button in Phase 1 (data is local; error is unlikely; a refresh resolves it)

## Interaction Contract

| Trigger | Action |
|---------|--------|
| Click trigger tab (collapsed) | Expand panel (translate-x-0), change tab icon to chevron-left, add backdrop |
| Click trigger tab (expanded) | Collapse panel (-translate-x-full), restore calendar icon, remove backdrop |
| Click backdrop | Collapse panel |
| Press Escape (panel open) | Collapse panel |
| Click quest item | Navigate to `/practice/:topicId?question_id=:questionId`; panel stays open |
| Click "Generate plan" | Navigate to `/review`; panel stays open |
| Click × close button | Collapse panel |

Default state on mount: **collapsed**. State is not persisted across page loads (reset to collapsed on each navigation).

## Accessibility Contract

- Sidebar panel: `role="complementary" aria-label="Study plan"`
- Trigger tab: `<button>` with dynamic `aria-label` and `aria-expanded` attribute
- Focus management: when panel opens, focus moves to the panel's close button
- When panel closes, focus returns to the trigger tab
- Quest items: `<button>` or `<a>` (navigating elements), not `<div>` with onClick
- Keyboard: Tab navigates through quest items; Enter/Space activates
- Escape closes panel (listener on `window`, removed on unmount)
- All interactive elements: `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-blue-500` (inherited from Button; apply manually to non-Button elements)
